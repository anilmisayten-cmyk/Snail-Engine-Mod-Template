Chapter 02

Mod Structure

Understanding the correct folder layout is critical. This matches the real base-game-main structure.

⚠️
IMPORTANT: Songs go in songs/ at the mod root — NOT data/songs/!
Folder Structure 
my-mod/
├── meta.json                    ← Mod metadata
├── data/
│   ├── characters/              ← Character .json + .hx scripts
│   ├── stages/                  ← Stage folders (data.json + script.hx)
│   ├── weeks/                   ← Week .json files
│   ├── noteskins/               ← Noteskin .json files
│   ├── notetypes/               ← Custom note types (.hx)
│   ├── events/                  ← Event definitions (.txt)
│   ├── dialogue/                ← Dialogue files
│   ├── scripts/                 ← Global scripts (pixelUI.hx, etc)
│   └── freeplay.json            ← Freeplay menu config
├── songs/                       ← 🎵 SONGS GO HERE!
│   └── my-song/
│       ├── audio/               ← Inst.ogg, Voices.ogg
│       ├── charts/              ← easy.json, normal.json, hard.json
│       ├── meta.json            ← Song metadata
│       └── info.txt             ← Legacy info
├── images/                      ← All visual assets
│   ├── backgrounds/             ← Stage backgrounds
│   ├── characters/              ← Character spritesheets (.png + .xml)
│   ├── cutscenes/               ← Cutscene animations
│   ├── menus/                   ← Menu assets
│   ├── pixelUI/                 ← Pixel art UI
│   └── UI/                      ← icon64, icons, notes
├── sounds/                      ← Sound effects
├── music/                       ← Background music
├── shaders/                     ← GLSL fragment shaders (.frag)
└── scripts/                     ← HScript files (global, modifiers, states)

Required Animations

Essential

idle · singLEFT · singDOWN · singUP · singRIGHT

Optional

singLEFTmiss · singDOWNmiss · singUPmiss · singRIGHTmiss · hey · danceLeft · danceRight

Spritesheet Formats

🖼️

PNG + XML (Sparrow)

Most common format. bf.png + bf.xml. Loaded via Paths.getSparrowAtlas().

🎬

Animate Atlas

Newer format. Folder with Animation.json + spritemap1.json + spritemap1.png.

📄

Packer Atlas

Simplest format. spirit.txt + spirit.png.

Chapter 04

Creating Songs

Songs live in the songs/ folder at mod root with audio, charts, and metadata.

⚠️
CRITICAL: Songs go in songs/my-song/ — NOT data/songs/!
Folder Structure 
songs/my-song/
├── audio/
│   ├── Inst.ogg               ← Instrumental (required)
│   ├── Voices.ogg             ← Vocals
│   ├── Voices-opp.ogg         ← Opponent vocals (optional)
│   └── Voices-player.ogg      ← Player vocals (optional)
├── charts/
│   ├── easy.json
│   ├── normal.json
│   ├── hard.json
│   └── events.json
├── meta.json                  ← Song metadata
└── info.txt                   ← Legacy format

Note Format

Each note in sectionNotes follows this format:

[strumTime, noteData, sustainLength]
strumTime — Note appearance time (ms)
noteData — Position: 0=Left, 1=Down, 2=Up, 3=Right
sustainLength — Hold length (0 = tap note)

Chapter 08

Scripting

Use Haxe scripts to customize gameplay, create modifiers, and control stages.

📜

Script Locations

scripts/global.hx — Global
scripts/myplaystate.hx — PlayState
data/stages/mystage/script.hx — Stage
scripts/modifiers/ — Modifiers
data/characters/*.hx — Character

🔧

Available Variables

FlxG, FlxSprite, FlxCamera, FlxTween, FlxEase, Paths, Conductor, PlayState, curBeat, curStep, bpm, scrollSpeed, songName

PlayState Script

scripts/myplaystate.hx 
function onCreate() {
    trace("Song: " + songName);
    trace("BPM: " + bpm);
}

function onBeatHit() {
    if (curBeat % 4 == 0) {
        FlxG.camera.zoom += 0.03;
    }
}

function onNoteHit(note) {
    FlxG.camera.shake(0.01, 0.1);
}

function onNoteMiss(note) {
    FlxG.camera.flash(FlxColor.RED, 0.3);
}

function onEndSong() {
    return 0; // 0 = continue, 1 = stop
}

Custom Modifier

scripts/modifiers/my-modifier.hx 
function getName() {
    return "myModifier";
}

function getModType() {
    return "NOTE_MOD"; // or "MISC_MOD"
}

function getOrder() {
    return 0;
}

🎨

Hue/saturation/brightness control per note.

Custom Shader

shaders/my-shader.frag 
uniform sampler2D uTexture;
uniform float uTime;
varying vec2 vTextureCoord;

void main() {
    vec4 color = texture2D(uTexture, vTextureCoord);
    color.r = sin(uTime) * 0.5 + 0.5;
    color.g = cos(uTime) * 0.5 + 0.5;
    gl_FragColor = color;
}

Installing Your Mod

How to load your mod into the game.

1

Place the mod folder

Copy your mod folder into the content/ directory of Snail Engine.

2

Launch the game

Open Snail Engine and wait for it to load.

3

Open Mods Menu

Press TAB to open the Mods menu.

4

Enable your mod

Find your mod in the list and toggle it on. Restart if needed.

FAQ

Frequently Asked Questions

Common questions and answers about Snail Engine modding.

Where do songs go?
In songs/ at the mod root — NOT data/songs/. Each song gets its own folder with audio/, charts/, meta.json, and info.txt.
Where do characters go?
In data/characters/ as .json files. Spritesheets go in images/characters/.
Where do stages go?
In data/stages/ — each stage gets its own folder with data.json and script.hx. Background images go in images/backgrounds/.
Where do scripts go?
Global scripts in scripts/. Stage scripts in data/stages/mystage/script.hx. Modifier scripts in scripts/modifiers/.
Where do shaders go?
In the shaders/ folder at mod root as .frag files.
Where do sound effects go?
Song audio in songs/my-song/audio/. Sound effects in sounds/. Background music in music/.
Where do images go?
Character sprites in images/characters/. Stage backgrounds in images/backgrounds/. UI elements in images/UI/.
How do I test my mod?
Launch the game, press TAB for the Mods menu, enable your mod, then play through Story Mode or Freeplay. Use the Chart Editor (debug menu) for testing songs.
